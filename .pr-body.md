## What's New

- **FastAPI backend** with full round CRUD API
- **Supabase integration** for persistent storage (replaces localStorage)
- **Frontend API client** (`api.js`) with offline fallback
- **Database migrations** for rounds table + pgvector extension
- **Auto-calculated stats**: score, putts, fairways hit, GIR
- **.env.example** for configuration

## API Endpoints

| Method | Path | Description |
|--------|------|-------------|
| POST | `/rounds` | Create new round |
| GET | `/rounds` | List rounds |
| GET | `/rounds/{id}` | Get round details |
| PATCH | `/rounds/{id}` | Update round |
| DELETE | `/rounds/{id}` | Delete round |
| POST | `/rounds/{id}/holes` | Add hole |
| POST | `/rounds/{id}/holes/{n}/shots` | Add shot |
| POST | `/rounds/{id}/finish` | Finish round |

## Setup Required

1. Run migrations in Supabase SQL Editor (`backend/migrations/`)
2. Copy `.env.example` to `.env` and add Supabase credentials
3. Install deps: `pip install -r requirements.txt`
4. Run: `uvicorn app.main:app --reload`

## Notes

- Auth is simple for now (`Bearer <user_id>`) — replace with JWT later
- Frontend syncs shots in real-time, falls back to localStorage if offline
- pgvector enabled for future RAG/LLM analysis

**Stack**: FastAPI + Supabase (Postgres/pgvector) + vanilla JS frontend
